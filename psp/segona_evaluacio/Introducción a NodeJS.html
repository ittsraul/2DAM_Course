1 Introducción

Nodejs es un runtime o entorno a ejecución construido sobre el motor JavaScript V8 de Google Chrome, y que nos permite hacer uso de este lenguaje para desarrollar cualquier tipo de aplicación, constituyendose como un entorno fullstack (frontend y backend) completo. Además, al tratarse de un lenguaje ligado inicialmente al mundo web, tenemos la posibilidad de crear aplicaciones de escritorio o móviles usando tecnologíes como HTML5 y CSS.

Al lugar Octoverse (https://octoverse.github.com/) creado por Github, se muestra un informe anual de las tendencias en cuanto a desarrollo de software libre, realizado a través de los millones de proyectos que se hospedan en este portal. En este informe, podemos ver como JavaScript es de lejos el lenguaje que más proyectos de software libre utilizan, seguido de Python en segundo lugar y de Java en tercero.

Documentación oficial Sitio web oficial con la documentación de nodejs: https://nodejs.org/en/docs/

1.1 Evolución de JavaScript

La primera versión de JavaScript fue diseñada el 1995 por Netscape, con la intención de dar inicialmente capacidad a los navegadores para la validación de formularios, sin necesidad de enviar los datos al servidor para ser validadas en tiempo donde la velocidad en las comunicaciones era muy limitada. Esta primera versión, inicialmente llamada LiveScript, pasó a nombrarse javaScript por puro marketing (java estaba de moda en la época).

El año 1997 se estandaritza el lenguaje, dando lugar a la norma ECMA-262 ( European Computer Manufactureros Association), conocido como ECMAScript. La ISO adoptó la ECMA-262 dando lugar a la estándar ISO/IEC-16262.

Desde entonces, el lenguaje ha ido evolucionando y ganando en capacidad, siendo la versión ES6 (EcmaScript 2015) la que mayores cambios supuso, con la introducción de ámbitos para las variables, string templates, funciones flecha, desestructuración, y sobredo, las clases y las promesas.

1.2 Características principales de Nodejs

Entre las características de nodejs, podríamos destacar:

  • Ofrece un paradigma de programación basada en eventos, a través de una API asíncrona que no bloquea el programa principal mientras espera una respuesta, sinò que sigue la ejecución principal hasta que se recibe la respuesta.
  • La ejecución de código es muy rápida,
  • Se basa en un modelo monohilo muy escalable: Únicamente hay un hilo de ejecución atendiendo peticiones, pero que gracias al API asíncrona y dirigida por eventos, se pueden atender múltiples peticiones por este hilo, llegando a consumir muchos menos recursos que un sistema multihilo.
  • Como que la ejecución no se realiza dentro del navegador, se elimina la necesidad de desarrollar código compatible con cada navegador.

1.3 Donde podemos encontrar nodejs?

Nodejs se encuentra seguramente en más lugares de los que podemos imaginar. Vemos una lista de aplicaciones donde se utiliza nodejs, tanto desde el lado de servidor como de aplicaciones de escritorio y móviles. Veamos algunos ejemplos:

  • Entornos de servidor (backend): Netflix, Paypal, LinkedIn, Ebay, NASA, Mozilla, Uber, Yahoo, Groupon (https://www.hackaboss.com/blog/5-mejores-ejemplos-de-aplicaciones-node-js-para-empresas),
  • Aplicaciones de escritorio:
    • Aplicaciones de escritorio basadas en Electron (https://electronjs.org/apps): Skype, Whatsapp, Slack, Atom, Visual Studio Code, Github Desktop
    • Aplicaciones basadas en nwjs (https://github.com/nwjs/nw.js/wiki/list-of-apps-and-companies-using-nw.js): Haroopad, Reditr, TweetDeck, Kindle Cloud Reader.
  • Aplicaciones móviles con Ionic/Capacitor: MarketWatch, Sanvello, Calla (https://mdevelopers.com/blog/14-best-ionic-apps-of-2022)

2 Primeros pasos con Nodejs

2.1 Instalación

En Ubuntu, nodejs ya vé instalado de serie, en su versión 10.19.0~dfsg-3ubuntu1 (Ubuntu 20.04 LTS) a fecha de diciembre de 2022.

Si deseamos la última versión, podemos descargar el paquete desde la propia web de nodejs, donde se encuentran las últimas versiones oficiales (18.12.1 LTS o 19.2.0 Current). La diferencia entre ambas es que la LTS se considera estable y es de soporte a largo plazo, mientras que la Current incorpora las últimas novedades, a cambio de perder un poco de estabilidad (podríamos decir que es una beta de la próxima versión LTS).

Si optamos para descargar esta versión, veremos que no se encuentra en forma de paquete, sino que es un fichero comprimido, por lo que haría falta desomprimir-lo y copiar cada cosa en su lugar. Por otro lado, al repositorio NodeSource de Github se encuentran unos scripts para facilitarnos la instalación desde el gestor de paquetes del sistema: https://nodejs.org/es/download/package-manager/#distribuciones-de-linux-basadas-en-debian-y-ubuntu. Lo que habrá que hacer en este caso será:

$ sudo apt-get update
$ sudo apt-get install curl
$ curl -sL https://deb.nodesource.com/setup_18.x | sudo -E bash -
$ sudo apt-get install -y nodejs

De todas formas, para el trabajo que realizaremos, nos será suficiente con la versión que lleva Ubuntu, de forma que no necesitemos pelear con dependencias a la hora de distribuir nuestras aplicaciones.

Lo que nos ofrece este paquete, tanto el que tenemos preinstalado al sistema como si lo instalamos de nuevo es el propio node (o su alias nodejs) y el gestor de paquetes de nodejs, denominado npm (Nodo Package Manager)

2.2 Hola Node

nodejs puede lanzarse desde la propia terminal, con la orden node o su àlias nodejs.

Para comprobar la versión de node que tenemos, haremos uso de node -v o node --version:

$ node -v
v18.12.1

nodejs admite dos formas de trabajo: mediante la consola interactiva y la ejecución directa de scripts.

2.2.1 La consola interactiva

Nodejs dispone de una consola interactiva, a la que podemos acceder simplemente con node y lanzar directamente órdenes:

$ node
> console.log("Hola Node");
Hola Node
undefined

Como podemos ver, cuando lanzamos node, el prompt del sistema cambia al símbolo >, indicando que estamos dentro de la consola de Node. Si alguna vez hemos trabajado con la consola web del navegador, el funcionamiento es exactamente el mismo.

Si nos fijamos, hemos escrito solo una instrucción console.log, que sería lo equivalen en Java a System.out.println. Podemos ver, como no nos ha hecho falta definir ninguna clase principal ni ningún método main. JavaScript, a pesar de que soporta la programación orientada a objetos, no obliga a que todo, incluido el programa principal esté dentro de una clase. De este modo, podemos tener código tan sencillo como una simple instrucción.

Vemos también que al final nos vuelve un valor undefined. Esto se debe a que la función console.log no devuelve ningún valor (es de tipo void), y por tanto, nos indica que el resultado que devuelve la función no está definido.

Para salir del modo interactivo haremos uso de la combinación de teclas Ctrl+D.

2.2.2 2.4. Ejecución de scripts

La manera habitual en que funcionaremos con nodejs será lanzando la orden node sobre un script. Veámoslo con un sencillo ejemplo. Tenemos el script siguiente, denominado hola.js.

if (process.argv.length==3)
 console.log("Hola "+process.argv[2])
else
 console.log("Hola Nodejs");

Para ejecutarlo, utilizremos node junto con el nombre del fichero:

$ node hola.js Jose
Hola Jose
$ node hola.js 
Hola Nodejs

Como vemos, se trata de un Hola Mundo personalizado, donde según los parámetros que le proporcionamos, escribirá una u otra cosa. También podemos observar, como se ha comentado antes, que no ha hecho falta incluir el código en ninguna clase ni función especial. Además, vemos que podemos acceder a los paràmetes que pasamos por la línea de órdenes a través del objeto (vector) process.argv, que la propiedad process.argv.length nos da la longitud de este, y que la segunda posición de este (process.argv[2]) hace referencia al primer parámetro que le hemos pasado.
