1 Introducción

Como hemos comentado, nodeJS se basa en el motor de JavaScript V8 de Google Chrome, por el que empezaremos con una introducción a este lenguaje para empezar con nodeJS.

Vemos algunas generalidades del lenguaje:

  • Un programa o script en JavaScript está compuesto de sentencias, que pueden o no acabar en ;. A pesar de que no es obligatorio, es recomendable su uso.
  • Los espacios en blanco sobrantes y las nuevas líneas, no se tienen en cuenta, por lo que admite cualquier tipo de tabulación.
  • Los bloque de código se definen siguiendo el estilo de lenguajes como C o Java, con llaves {...}.
  • Se trata de un lenguaje case sensisive, por lo que distingue mayúsculas y minúsculas.
  • Los comentarios también siguen una sintaxis similar a C o Java:
  • // Dos barras para los comentarios de una línea
  • /* Barra y asterisco para comentarios de más de una línea */
  • Palabras reservadas: Aquellas que se utilizan para construir sentencias, y que no se pueden utilizar libremente. Alguens de estas son: break, caso, catch, class, continúo, default, delete, do, else, finally, for, function, if, in, instanceof, new, return, switch, this, throw, try, typeof, var, void, while, with.
  • A pesar de que JavaScript soporta programación orientada a objetos, no estamos obligados a utilizarlos. Por lo tanto, no es necesario definir un método Main dentro de una clase, sinò que directamente la ejecución empieza con la primera línea de código ejecutable.

2 Elementos del lenguaje

2.1 Variables

No es necesario declarar variables en JavaScript, a pesar de que es altamente recomendable. Para declarar una variable hacemos:

let variable=valor;

También se puede utilizar var en lugar de let, pero desde la versión ES2015, la recomendación es utilizar let.

2.1.1 El modo strict

JavaScript nos permite trabajar en modo strict, que nos obliga a declarar las variables antes de utilizarlas.

Por ejemplo, el siguiente script (saluda.js):

nombre="Jose";
console.log("Hola "+nombre);

Funcionaría correctamente:

$ nodo saluda.js
Hola Jose

Pero si habilitamos el modo estricto:

'uso strict';

nombre="Jose";
console.log("Hola "+nombre);

Nos mostraría el siguiente error:

/tmp/prova1.js:3
nombre="Jose"

ReferenceError: nom is not defined
    at Object.<anonymous> (/tmp/saluda.js:3:4)
    at Module._compile (module.js:652:30)
    at Object.Module._extensions..js (module.js:663:10)
    at Module.load (module.js:565:32)

Como vemos, el trabajar con el modo strict, nos obliga a definir variables, de forma que detectamos fácilmente cuando cometemos algún error escribiendo el nombre de estas.

2.1.2 El nombre de las variables

  • El nombre de las variables puede contener cualquier carácter alfanumérico, más el símbolo $ y _.
  • Pueden empezar por cualquier carácter, más el $ y el _,pero no por un número.
  • El nombre de las variables suele indicarse con formato CamelCase (la primera letra de cada palabra en mayúsculas y el resto en minúsculas).

2.1.3 Tipo de variables y asignación dinámica

Una variable también puede ser inicializada en la declaración, y puede ser cualquier valor, o el resultado de una expresión.

let variable=[ valor | expresión ];

Una variable en javascript almacena datos de un tipo concreto en un momento dado. Mediante la asignación dinámica de tipo, el tipo de dato que almacena esta variable puede cambiar en tiempo de ejecución. Es decir, el tipo de una variable se define de forma dinámica en el momento de su asignación, no de la declaración.

Las variables pueden ser:

  • Variables de tipos numéricas (number): Almacena valores enteros o decimales (con el carácter . para separar la parte entera de decimal).
  • Tipo texto (string): Almacena una lista de caracters, entre comillas, bien con comillas simples o dobles. En caso de que el texto contenga comillas, hay que tener cuidado con la construcción, no pudiendo utilizar internamente las comillas que hemos utilizado para definir la cadena de texto o bien escapando estas:
var texto="Esta frase utiliza 'comillas simples'"
var texto="Esta frase utiliza \"comillas dobles escapadas\""
  • Dentro de las cadenas de texto, también podemos incluir:
    • Saltos de línea: \n
    • Tabuladores: \t
    • Barra inclinada: \\
  • Comillas escapadas: \" o \'

Conversión de tipo

La conversión de tipo se realiza de forma automática. En caso de ambiguedad, utiliza prioridades:

Ejemplo: '10'+3 : Suma un string y un number. En este caso, tiene prioridad la cadena de caracteres, por lo que el number 3 se convierte al string ‘3’. En cambio, cuando se trata del + unario, solo se aplica a los number, por lo que +‘20’, representaría el número 20.

Veamos algunos ejemplos más:

13 + 7 => 20
"13" + "7" => "137"
"13" + 7 => "137"
+"13" + 7 => 20

Tipo lógico: Pueden contener los valoras true y false.

let itemSelected=false;
let display=true;
  • Valores null y undefined: Cuando una variable no se ha definido, el tipo de esta es undefined. En cambio, cuando queremos hacer referencia al valor nulo, hacemos uso de null, sin comillas. Como veremos, el valor null será un tipo especial de objeto.

Typeof

Para saber el tipo de dato que contiene una variable en un momento dado, podemos utilizarl operador typeof:

> typeof(10)
'number'
> typeof("hola")
'string'
> typeof(true)
'boolean'
> typeof(a)
'undefined'
> typeof(null)
'object'
> typeof({"nombre":"jose"})
'object'

2.1.4 Vectores y JSON

Los vectores son colecciones de valores/elementos, bien del mismo tipo o de tipo diferente (a diferencia otros lenguajes, que los tipos tienen que ser homogéneos).

Para definir el vector utilizamos los caracters [ y ] al principio y al final, y utilizamos las comas (,) para separar elementos.

let nombre_vector = [valor1, valor2, ..., valorN];

Ejemplos:

let días_semana=["lunes", "martes", "miércoles", "jueves", "viernes", "sábado", "domingo"];
let ex_vector=["cadena1", num1, "cadena2", ["cadena1_vector1"]];

Para acceder a los elementos, indicamos la posición en el vector (la posición inicial es la 0):

  • Consulta: var hoy=día[1];
  • Asignación: ex_vector[1]="cadena3";
  • Un vector vacío, se puede definir inicializándolo a vacío, o creando un nuevo objeto de la clase Array:
ex_vector2=[];
ex_vector3=new Array();

Por su parte, los elementos JSON (JavaScript Object Notation) son una forma más compacta de almacenar y transmitir información que el formato XML, a través de pares clave-valor.

Los los tipos de datos que podemos representar en un JSON son:

  • Números, tanto enteros como decimales,
  • Cadenas, expresadas entre comillas y con la posibilidad de incluir secuencias de escape,
  • Booleanos, para representar los valores true y false,
  • Null, para representar el valor nulo,
  • Array, para representar listas de cero o más valores, de cualquier tipo, entre corxets y separados por comas,
  • Objetos, como colecciones de pares <clave>:<valor>, separados por comas y entre claves, y de cualquier tipo de valor.

Recordamos un pequeño ejemplo de esta notación:

{
 "curso": [
 {
 "nombre": "Acceso a Datos",
 "horas": 6,
 "qualificacio": 8.45
 },
 {
 "nombre": "Programación de servicios y procesos",
 "horas": 3,
 "qualificacio": 9.0
 }
 ]
}

Javacript soporta la manipulación de elementos JSON de forma nativa, de modo que no necesitamos ninguna librería para poder utilizar este formato.

2.1.5 Declaración de constantes

Cuando no deseamos que cambie el valor de una variable o si queremos definir un valor global inmutable, se recomienda utilizar la palabra reservada const en lugar de let. Para las constantes globales se recomama usar mayúsculas.

const PI=3.1416;

2.2 Operadores

Las variables se combinan entre ellas, formando expresiones mediante los operadores. Estas expresiones, podrán evaluarse, dando lugar a un nuevo valor, que puede utilizarse para tomar decisiones en un script, o bien para asignarlo a nuevas variables.

Los operadores pueden ser de asignación, aritméticos, lógicos o relacionales.

2.2.1 Operador de asignación (=)

Se utiliza para asignar a una variable un valor o expresión.

let variable1 = Expresión;

2.2.2 Operadores aritméticos

Los operadores artimètics son aquellos que trabajan con valores de tipos numéricos para obtener otros valores de tipo también numérico. Además de los habituales de suma(+), resta (-), multiplicación (), división (/), y resto de la división (%), tenemos los siguientes operadores:

  • Operadores de incremento (++) y decremento (–): para incrementar o decrementar el valor de una variable, y no tiene el mismo significado ponerlo delante que detrás de la variable.
let a=2
let b=2
let c=++a + b;
>> c = 5
>> a = 3

No es el mismo que:

a2=2
b2=2
c2=a2++ + b2
>> c2 = 4
>> a2 = 3
  • Suma/resta Combinada (+= y -=): Al igual que en C y Java, podemos utilizar expresiones tipos a+=3 para simplificar la expresión a=a+3.

2.2.3 Operadores lógicos

Son aquellos que se aplican sobre valores lógicos y devuelven un valor también lógico.

  • Negación: !
> let a=true;
> !a;
false

Cuando se aplica sobre valores no lógicos el significado es el siguiente: El valor numérico de 0 y la cadena de texto vacía se convierten al valor lógico false, mientras qu eun número diferente a 0 y una cadena no vacía se consideran true.

  • Operador and (&&): El resultado es cierto si los dos operandos lo son. Operador oro (||): El resultado es cierto si uno de los dos operandos lo es.

2.2.4 Operadores relacionales

Son aquellos que operan con cualquier tipo de dato, comparándolos y nos proporcionan un valor de tipo lógico.

Operador Semántica
> Mayor que
< Menor que
>= Mayor o igual
<= Menor o igual
== (Cuenta con =) Igual que
5=="5" ==> true
=== Igual que, en valor y tipo.
5==="5" ==> false
!= Distinto de
!== Distinto, o de tipo distinto

Cuando trabajamos con cadenas, los operadores relacionales lo que hacen es comparar letra a letra, los caracteres ASCII de dos cadenas, de izquierda a derecha.

2.2.5 Otros operadores

Además de los operadores tradicionales, javascript aporta algunos operadores más:

  • Operador condicional ternario: El operador condicional ternario, permite asignar un valor a una variable, basándose en cierta condición: el
variablename = (condition) ? value1:value2

Por ejemplo:

let mayor_edad = (edad < 18) ? "Menor de edad":"Mayor de edad";
  • Operador Typeof: Como ya hemos comentado antes, este operador obtiene el tipo de valor de una variable.

3 Funciones

La delcaración de una función en JavaScript se declara con la palabra reservada function. A diferencia de Java, las funciones pueden declararse fuera de un objeto.

function nombre_funcion(lista_argumentos) {
 // Cuerpo de la función
 }

El nombre de la función se utiliza para hacer referencia a ella desde cualquier parte del código. Si queremos que la función nos devuelva algún valor, y asignar este a una variable, haremos uso de la palabra reservada return. La orden de los argumentos es importante (paso de parámetros posicional). Además, el número de las variables utilizadas cuando se invoca la función no tiene por qué coincidir con el número que le damos a los argumentos en la declaración.

Ejemplo:

function suma(numero1, numero2) {
 resultado = numero1 + numero2;
 return resultado;
}

s=suma(a, b);

También hay que recordar que una función devuelve únicamente un valor, por lo que en el caso que necesitamos devolver información variada, podemos utilizar JSON.

Ejemplo:

function calculaPrecio(precio) {
    precio_iva=(precio*1.21).toFixed(2); // Redondeamos a dos decimales
    precio_iva_reduicido=(precio*1.1).toFixed(2); 
 return {"precio_con_iva": precio_iva, "precio_iva_red":precio_iva_reducido}
}

let precios = calculaPrecio(50);
console.log(precios);
>> Object { precio_con_iva: "60.50", precio_iva_red: "55.00" }

3.1 Ámbito de las variables (scope)

El ámbito de una variable es la parte del programa donde esta está definida, y dónde toma sentido. El ámbito puede ser:

  • global, visible desde todo el script, o * local, visible solo en la función donde se encuentra declarada (con var o let).

Para entender bien el ámbito de las variables, hay que tener en cuenta algunos aspectos:

  • JavaScript es un lenguaje con ámbito global como ámbito predeterminado, y donde todo se pasa también de forma predeterminada por referencia: cuando declaramos una variable fuera de una función, esta es global, y se pasa por referencia a los ámbitos descendentes o heredados (por lo tanto, cualquier modificación que hacemos a la variable en cualquier ámbito tendrá efecto sobre ella).
  • Cuando declaramos una variable con var dentro de una función, se crea un variable local, con visibilidad restringida a esa función.
  • Cuando definimos una variable con let o una constante con const, su ámbito se reduce a los bloques donde está definida.

Vemos-lo con un ejemplo. El siguiente código define algunas variables, con let y var en diferentes ámbitos:

var var2=2;

function fun(){

        else console.log("var1 no està definida");
    if (typeof(var2)!="undefined") console.log("var2="+var2);
        else console.log("var2 no està definida");
    if (typeof(var3)!="undefined") console.log("var3="+var3);
        else console.log("var3 no està definida");
    if (typeof(var4)!="undefined") console.log("var4="+var4);

        else console.log("var5 no està definida");
    if (typeof(var6)!="undefined") console.log("var6="+var6);
        else console.log("var6 no està definida");

}

    else console.log("var1 no està definida");

    else console.log("var2 no està definida");

Parémonos un momento a pensar la salida que tendrá este script, y una vez lo hayamos pensado, veamos el resultado para contrastar:

Algunas observaciones:

  • Dentro de la función, tenemos acceso a todas las variables definidas, tanto fuera, como dentro, a excepción de la variable var6, que se ha definido dentro de un bloque de código con la palabra reservada let.
  • Dentro de la función, var1 y var2 han modificado su valor, pero fuera de la función, la modificación se ha mantenido sólamente para var1, no viéndose modificada var2. Esto se debe a que la modificación realizada sobre var1 se ha hecho sobre la referencia a la variable global que se ha pasado a la función, mientras que var2 se ha definido de nuevo con var dentro de la función, siendo, por lo tanto, una referencia diferente a la var2 definida globalmente.
  • Y finalmente, vemos como todas las variables que se han definido dentro de la función, independientemente de si se han hecho con var o let, no tienen visibilidad fuera de la función.

La recomendación es siempre utilizar la palabra reservada let, y definir las variables siempre de forma local a aquellos ámbitos donde se necesitan. Cuando necesitamos pasar variables a una función, lo haremos siempre por el paso de argumentos.

4 Programación estructurada en JavaScript

4.1 Estructuras condicionales

4.1.1 if

Se trata de la condición más sencilla:

if (condición) {
 // bloque de código a ejecutar si la condición se evalúa a cierto
}

Ejemplo 1

if (hora < 12) {
 saludo = "Buenos días";
}

4.1.2 if / else

if (condición) {
 // bloque de código a ejecutar si la condición se evalúa a cierto
} else {
 // bloque de código a ejecutar si la condición se evalúa a falso
}

Ejemplo

if (hora < 12) {
    saludo = "Buenos días";
} else {
    saludo = "Buenas tardes";
}

4.1.3 else if

Cuando pueden darse varias condiciones alternativas, podemos utilizar la estructura else/if.

if (condición1) {
    // bloque de código a ejecutar si la condición 1 se evalúa a cierto
} else if (condición2) {
    // bloque de código a ejecutar si la condición 2 se evalúa a cierto
} else {
 // bloque de código a ejecutar si ninguno de las dos condiciones se avalúa a cierto.
}

Ejemplo

if (hora < 12) {
 saludo = "Buenos días";
} else if (hora < 21) {
 saludo = "Buenas tardes";
} else {
 saludo = "Buenas noches";
}

4.1.4 Switch

Se utiliza cuando queremos realizar diferentes acciones en función de múltiples condiciones sobre una expresión. Según esta se evalúe, se ejecutará uno (o más, cuidado con el break!) de entre varios bloques.

switch(expresión) {
 case n1:
    // Bloque de código a ejecutar si expresión se evalúa a n1
 break;
 case n2:
    // Bloque de código a ejecutar si expresión se evalúa a n2
 break;
 default:
    // Bloque de código a ejecutar si la expresión se evalúa
    // a un valor no contemplado
}

Ejemplo: Usando la función Date().getDay() que devuelve el día de la semana:

switch (new Date().getDay()) {
    case 0:
        dia = "Diumenge";
        break;
    case 1:
        dia = "Dilluns";
        break;
    case 2:
        dia = "Dimarts";
        break;
    case 3:
        dia = "Dimecres";
        break;
    case 4:
        dia = "Dijous";
        break;
    case 5:
        dia = "Divendres";
        break;
    case 6:
        dia = "Dissabte";
        break;
    default:
       console.log("dia erroni");    }

El break se utiliza para indicar que ya no hay que seguir buscando más coincidencias. Ahorramos así comparaciones innecesarias y errores. Hay que tener en cuenta que también se pueden agrupar diversos caso (sin el break por el medio), cuando hay código común.

Ejemplo:

switch (new Date().getDay()) {
    case 1:
    case 2:
    case 3:
    case 4:
    case 5:
        dia = "dia laboral";
        break;
    case 0: // Diumenge
    case 6: // Dissabte
        dia = "cap de setmana";
        break;
    default:
       console.log("dia erroni");
}

4.2 Estructuras repetitivas

4.2.1 Bucle for

Se usa generalmente cuando sabemos de antemano cuantas repeticiones necesitamos:

for (inicialización; condición_de_repetición ; incremento ) {
    // Código a ejecutar
}

Ejemplo:

for (i = 0; i < 10; i++) {
    console.log("Indice: " + i); // Fijáos que "sumamos" un string y un entero!
}

Sintácticamente la inicialización, condición de repetición y el incremento no son necesarias, y el bucle anterior podría expresarse como:

var i=0;
for(;;){
 console.log(i++);
 if (i>10) break; // El break rompe el flujo del bucle
}

Por otro lado, también podemos inicializar varios valores:

****Ejemplo****: Para mostrar los elementos de un vector:

vector=["pera", "manzana", "plátano"];
for (i = 0, len = vector.length, texto = ""; i < len; i++) {
    console.log(vector[i]);
}

4.2.2 Bucle for/in - for/of

Se utiliza para recorrer los elementos de un objeto:

for(item in object){
    ...
}

Ejemplo:

let peli={title:"Han Solo: Una historia de Star Wars", director: "Ron Howard"};

for (index in peli) {
    console.log(index+":"+peli[index]);
}

Así también podemos indexar los elementos de un vector:

 v=[1,2,3,76,4,2,5]
 ...
> for (i in v) {
... console.log(v[i]);}

Un error habitual (por influencia otros lenguajes de script) es confundir el índice con el valor del elemento:

for (e in v) {
 console.log(e);
} // Cuando el que queremos es acceder al contenido!

Para acceder directamente a los elementos, lo que nos ofrece JavaScript es el operador of. El único requerimiento es que el objeto sobre el que lo aplicamos sea iterable.

Por ejemplo, el siguiente código que imprime los elementos del vector mediante for..in:

> for (i in v) {console.log(v[i])};

Puede expresarse más sencillamente mediante for..of con:

> for (i of v) {console.log(i)};

4.2.3 Bucles While y do/while

Son equivalentes al bucle for, pero con una sintaxis diferente, donde tenemos que realizar tanto la inicialización como el incremento fuera de la sintaxis de la instrucción.

While

inicialitzaió;
while(condición){
    // acciones;
    //incremento;
};

Do - While

// [inicialización;]
do{
    //acciones;
    //[inicialitació]

} while (condición)

Ejemplos:

let i=1;
while(i<10){
   console.log(i);
   i++;
}
let i=1;
do{
     console.log(i);
     y++;
   } while (i<10);
}

4.2.4 Break y continue

El break, como hemos visto, sirve para salir de un bucle (y también de uno case). Por su parte, la orden continue, sirve para pasar a la siguiente iteración del bucle.

Ejemplo

let i=0;
for(;;){
    i++;
    if(i%2) continue; // A què se evalua i%2 ?
    console.log(i);
    if (i>=10) break;
}

5 Programación Orientada a Objetos en JS

En el paradigma de orientación a objetos, el software se diseña a través de objetos que cooperan, intercambiándose mensajes y respondiendo a events, a diferencia del punto de vista tradicional donde se considera como un conjunto de funciones.

Veamos algunos conceptos:

  • Clase: Define las características del objeto.
  • Objeto: Una instancia de la clase.
  • Propiedad/Atributo: Una característica (variable) del objeto.
  • Método: Una capacidad (función) del objeto.
  • Constructor: Método que se invoca al crear el objeto.
  • Herencia: Capacidad de una clase de heredar características de otra clase.
  • Encapsulamiento: Las propiedades y los métodos de los objetos se encuentran organizados de manera estructurada, evitando el acceso desde cualquier forma distinta a las especificadas.
  • Polimorfismo: Mecanismo por el cual diferentes clases pueden implementar de forma diferente un mismo método.

5.1 Orientación a Objetos basada en prototipos

Se trata de un estilo de programación orientada a objetos donde estos no se crean a través de la instanciació de clases, sino mediante la clonación otros objetos que hacen de prototipos. También se conoce como programación sin clases, orientada a prototipo o basada en ejemplos.

5.1.1 Espacios de nombres

En aplicaciones grandes, solemos utilizar los espacios de nombres para evitar conflictos de nombres con entre los diferentes scripts y librerías que las componen. Estos actúen como contenedores que permiten asociar la funcionalidad y las propiedades de un objeto con un nombre único y aislado del resto. (Podría equipararse a los packages de Java).

En javascript no existe una sintaxis nativa para esa funcionalidad, sinò que se utiliza un objeto para esta finalidad. Por ejemplo:

var app={}
app.variable1 = 'variable1'
app.variable2='variable2'
app.funcion = function () { .... }

La definición de un objeto prototipo en javascript no es más que una función, que hace al mismo tiempo de constructor:

function Objeto() { 
    // Constructor e inicialización de valores
    }

Y para crear un nuevo objeto basado en este prototipo, hacemos uso de new:

var obj=new Objeto();

Cuando se define así un objeto, este tiene acceso a una propiedad especial, denominada prototipo (prototype), que permite acceder a la clase en sí.

Para acceder a propiedades dentro de una clase se utiliza la palabra reservada this. Habrá que tener cuidado con esta, puesto que cuando tenemos objetos o propiedades anidades, this siempre hará referencia al objeto del nivel donde se encuentra.

Para acceder desde fuera de la clase a un atributo o método hacemos uso de la notación punto (.): Objeto.propiedad / Objeto.método.

Por ejemplo, para definir la propiedad nombre de una clase persona, podemos hacerlo en la creación de la instancia:

function Persona(nombre) {
    this.nombre = nombre;
}

Y para instanciar dos nuevas personas:

let persona1 = new Persona("Paco");
let persona2 = new Persona("Pepe");

console.log ('persona1 es ' + persona1.nombre); // muestra "persona1 se Paco"
console.log ('persona2 es ' + persona2.nombre); // muestra "persona2 se Pepe"

Para incluir métodos, lo que hacemos es definir funciones adentro la clase:

function Persona(nombre) {
 this.nombre = nombre;
 this.saluda=function(){
        alert('Hola, me llamo '+this.nombre);
    }
}

var persona1=new Persona('Paco');
persona1.saluda();

Cómo vemos, los métodos no son más que funciones que se asocien como una propiedad al objeto, de forma que pueden invocarse fuera de su contexto.

Ejemplo

Veamos un ejemplo completo para la definición de clases, y que nos servirá para ver otros conceptos como el de la herencia:

/* Exemple de definició de classes */

function punt(x, y){
    /* 
         Funció constructor de la classe punt
       Rep dos paràmetres amb els que inicialitza 
       els atributs x i y
     */

    this.x=x;  // Definim atributs amb la paraula 
    this.y=y;  // reservada "this"

    this.get=function(){
        // Els mètodes no són més que funcions
        // Associades a la classe, i que definim
        // amb this, igual que un atribut.
        return "("+this.x+","+this.y+")";
    }
}

function figura(color, position){
    /*
        Classe figura, té com atributs un color
        i una posició (objecte de tipus punt)
    */

    this.color=color;     // Inicialitzem el color
    this.position=position; // Inicialitzem la posició

    this.draw=function(){
        /*
              Mètode que mostra un misstge donant 
          informació de la figura
        */
        console.log("Dibuixant figura en posició "+this.position.get()+
                      " i color "+this.color);
    }
}

function rectangle(color, posicio, costat1, costat2){
    
    /*
    Classe rectangle, que és una especialització de 
        la classe figura. De figura hereta:
        - l'atribut posició 
        - l'attribut color
    A més:
        - Inclou dos nous atributs: costat1 i costat2
        - Redefineix el mètode draw
        - Defineix un nou mètode area
    */

    this.costat1=costat1;
    this.costat2=costat2;
    /*
    Per heretar propietats de figura, 
    fem ús de "call", passant this com a 
    primer paràmetre, i els valors amb què
    volem invocar al constructor de la
    superclasse
    */

    figura.call(this, color, posicio);

    this.draw=function(){
        console.log("Dibuixant Rectangle de "+
                 this.costat1+"x"+this.costat2+
                 " en posició "+this.position.get()+
                 " i color "+this.color);       
        };

    this.area=function(){
        return this.costat1*this.costat2;
        }
    }


function cercle(color,posicio, radi){
    /*
    Classe cercle, hereta de figura: posició i color
    Afig l'atribut radi
    I redefineix el mètode draw i defineix area
    */

    this.radi=radi;
    // Apply és pot utilitzar també per heretar de la superclasse
    // La diferència és que requereix el pas de paràmetres a través
    // d'un vector:
    figura.apply(this, [color, posicio]);

    this.draw=function(){
        console.log("Dibuixant Cercle de radi "+this.radi+
                       " en posició "+this.position.get()+
                       " i color "+this.color);
        };

    this.area=function(){
        return 2*this.radi*Math.PI;
        }
    }

p1=new punt(10,20);
p2=new punt(10,30);
p3=new punt(30,20);

f1=new figura("roig", p1);
f2=new rectangle("verd", p2, 40, 20);
f3=new cercle("blau", p3, 50)
f1.draw();
f2.draw();
f3.draw();

5.2 Orientación a Objetos a partir de ES6

A partir de la versión EcmaScript 6, JavaScript soporta una sintaxis para la orientación a objetos más próxima a la del resto de los lenguajes. Hay que remarcar que no se trata de una reformulación del modelo de objetos, sinò que simplemente se dedica a proveer de definiciones más claras y simples para el trabajo con objetos.

5.2.0.1 Declaración de clases

Como en otros lenguajes, se puede declarar utilizando la palabra class:

class Clase{

}

let objeto = new Clase;

Hay que tener en cuenta algunos detalles:

  • La clase no necesita de argumentos para su definición, por lo que puede no se acompaña de los paréntesis(new Clase en lugar de new Clase()).
  • Dado que se ha definido como clase, el sistema no permitirá que se ejecute como función (a diferencia de los prototipos), sinò que se reservará como un constructor.
  • El contenido de la clase se ejecuta en modo "strict" de manera automática.
  • Las declaraciones de clase no siguen las reglas del *hoisting* (la declaración puede ser posterior al uso). Esto significa que las clases solo existen después de ser declaradas.
  • Una clase se comporta implícitamente como una constante, por lo que no se puede redeclarar más adelante en un mismo ámbito.

Veamos la declaración de clase de forma más ampliada:

class  Classe{
    constructor (arguments){
        // super (arguments); 
        // Declaració d'atributs amb this
    }

    get getAtribut(){
        // getter; compte que el nom no siga el
        // mateix que l'atribut

        return (this.atribut);
    }
    set setAtribut(valor){
        // setter; compte que el nom no siga el
        // mateix que l'atribut
        this.atribut=valor;
    }

    static staticMethod(){
        // Aquest és un mètode estàtic,
        // S'exectarà des de la classe, 
        // no des de les instàncies d'aquesta.
    }

    Metode(){
        // Mètode públic
        // No existeixen per tant els privats o protegits
    }
}

Vemos otro ejemplo más completo:

class Classe{
constructor(valor=1){
    // Atributs de la classe (_a)
    // Si no s'especifica, el valor per defecte és 1
    this._a=valor;
}

get a(){
  // getter (no s'ha de dir igual que l'atribut!)
  return this._a;
 }

}
// Instanciació sense especificar valor
b=new Classe;
console.log(b.a);

// Instanciació amb valor
c=new Classe(4);
console.log(c.a);

El resultat de l’script serà:

1
4

5.2.0.2 Herencia

La herencia se consigue mediante la parala clave extends:

class Sublclasse extends Superclasse{
    constructor(params={}){
        // Invocació al constructor de la classe pare
        super(valors);
    }

    metode(){
        // Invocació a mètodes de la classe pare
        super.metode();
    }
}

Como podemos ver, con la palabra reservada super podemos acceder a las propiedades y métodos del padre. Si la utilizamos en forma de función (super(valores)) invocamos al constructor de la clase padre, con los valores por defecto que le pasamos. Por su parte, cuando la utilizamos en forma de objeto (super.metodo()), lo que hacemos es invocar a un método de la clase padre.

Algunos aspectos a destacar sobre la herencia en JavaScript:

  • El constructor de una clase hija, necesariamente tiene que invocar a super antes de utilizar this.
  • Los constructores de las subclases, tienen que invocar a super, para inicializar la clase, o bien devolver un objeto que reemplace el objeto que no ha sido inicializado.

Vemaos finalmente como quedaría el ejemplo anterior con esta nueva sintaxis:

        this.costat2=costat2;

    console.log("Dibuixant Rectangle de "+
               this.costat1+"x"+this.costat2+

    }

    area(){
        return this.costat1*this.costat2;
        }
